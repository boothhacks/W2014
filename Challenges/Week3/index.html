Week 3 Challenge

For this challenge, you'll need to do a git pull to get the latest files from our github repo.

Once you do, you'll need to search through the boothhacks/W2014/Searchme directory and count the total number of lines that contain the word "Job" in files that contain the word "apply" in their name.

For example, the following files would each count:

file name file contents notes
Searchme/apply.txt Job
Searchme/123apply456.txt abcJobdef It still counts if there's additional text in the file name or file contents
Searchme/Folderxyz/apply.txt Job Any file counts, no matter how deep its burried underneath directories
Searchme/apply.txt abcJobdefJobhij Even if "Job" appears twice, it only counts as 1 line
Searchme/apply.txt abcJobdef
Jobhij		 However, if "Job" appears on 2 seperate lines, then that counts as 2

While these files would NOT match:

file name file contents notes
somename.txt Job file name doesn't match the string "apply"
apply.txt blahblah file contents doesn't match string "Job"
apply.txt job lowercase "job" doesn't count!

Go to this Google Doc to fill out your name and your guess. Whoever is the first person to get it right will get the (small) prize (to break ties, we'll look through the google doc revision history to see who put it first).

Good Luck!

Hint: we didn't talk about exactly how to do this last week. We talked about grep and find, but we didn't talk about how to count the number of lines, and we didn't talk about how to grep the contents of files outputted by find (it's not as simple as piping the output!)
